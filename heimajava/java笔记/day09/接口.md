# 接口定义

接口就是多个类的公共规范。接口是一种引用类型，最重要的是其中的抽象方法。

定义一个接口：public interface 接口名称{	//接口内容	}

文件后缀名仍是.java,编译后仍是.class

接口不能有静态代码块

接口不能有构造方法

# 接口包含的内容

1.常量

2.抽象方法

3.默认方法

4.静态方法

5.私有方法

# 接口中抽象方法

## 定义

public abstract 返回值类型 方法名称(参数列表);

## 注意：

1、接口中的抽象方法，修饰符必须是两个固定的关键字：public abstract

2、这两个关键字可以选择性省略

```java
public interface DemoInterfaceAbs {
    //这是一个抽象方法
    public abstract void method1();
    //这也是一个抽象方法
    abstract void method2();
    //这也是一个抽象方法
    void method3();
}
```

# 接口的使用

1、接口不能直接使用，必须有一个实现类来实现该接口。

格式：public class 实现类名称 implements 接口名称{...}

2、接口的实现类必须覆盖重写（实现）接口中所有的抽象方法；除非实现类是抽象类

实现：去掉abstract关键字，加上方法体大括号。

3、创建实现类的对象，进行使用。

# 接口中默认方法

## 定义

格式：public（必须是public，但可以省略） default 返回值类型 方法名称（参数列表）{ 方法体 }

默认方法可解决接口升级问题。比方说，要在接口中添加新方法，但是在实现类中也必须要实现这一方法，不然会报错，但是如果实现类较多这样做会比较麻烦，所以可以将新添加的方法设置为默认方法。

## 使用

1、接口的实现类“继承“了接口中的默认方法。可以在实现类中直接调用默认方法。

2、接口的实现类也可以覆盖重写接口的默认方法

# 接口中静态方法

## 定义

public static 返回值类型 方法名称（参数列表）{ 方法体 }

## 使用

接口名.静态方法名（参数）

不能通过接口实现类的实例来调用接口的静态方法，因为一个类可以实现多个借口，方法名可能一致，会产生混淆。

# 接口中私有方法

## 定义

为了解决接口中代码重复问题，同时又不希望实现类可以访问该方法，于是可以定义一个私有方法。

### 普通私有方法

解决多个默认方法之间代码重复的问题

格式：private 返回值类型 方法名称（参数列表）{ 方法体 }

### 静态私有方法

解决多个静态方法之间代码重复问题

格式：private static 返回值类型 方法名称(参数列表){ 方法体 }

## 使用

只能本接口中的默认方法或是静态方法才能调用私有方法。

# 接口中常量

## 定义

接口当中也可以定义“成员变量”，但是必须使用public static final关键字修饰，从效果上看，这其实就是常量。

格式：public static final 数据类型  常量名称  =   数据值

public static final可以选择性省略。常量必须进行赋值。常量名称大写，用下划线分割

## 使用

接口名称.常量名称

# 继承父类并实现多个接口

实现类实现了多个接口，并且接口类中有重名的抽象方法，只要覆盖重写其中一个即可

如果实现类没有重写所有接口类中的所有抽象方法，则该实现类必须是抽象类

如果多个接口类当中存在重名的默认方法，则实现类必须重写该默认方法（重写方法不需要default关键字）

一个实现类直接父类中的方法和接口类中的默认方法冲突，优先使用父类方法

# 接口之间的多继承

类与类之间是单继承的，直接父类只有一个

类与接口是多实现的，一个类可以实现多个接口

接口与接口是多继承的。

多个父接口的抽象方法重复，不会报错

多个父接口的默认方法重复，子接口必须重写该默认方法（default不能省略）
